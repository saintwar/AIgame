### A1. 字体子集补"电"字（hotfix）
**现象**：splash 加载页 "请使用台式电脑进行游戏"，"电"字回退系统字体，字形不一致。
**根因**：5/27 子集化时该提示尚未加入；提示加入后没人重跑 `build-font-subset.py`。
**修复**：
- 重跑子集脚本，woff2 305KB → 331KB（多 26KB / +80 个汉字）
- 字体 URL 从 `?v=20260527` 统一 bump 到 `?v=20260605`（preload + 两个 @font-face 全部改）
- **副收益**：之前因为 preload 用旧版本号，浏览器**实际加载了两份字体**（636KB），现在统一后只下载一份（331KB）。console 警告 `was preloaded ... but not used` 也消失。

## 2. 战斗 UX 升级

### A8. 拉力红区指针强化（方案 A + D）
**痛点**：原"快收线/快放线"屏幕中央文字反应链路太长（眼睛→中央→识别→动作 0.5s+），玩家来不及看。
**方案**：
- **A 拉力条本身视觉强化**：
  - 红区 ▲ 指针放大 50%（半宽 6→9 / 高 8→12）
  - 左红橙红 `#E67E22` / 右红正红 `#E84C3C` / 4Hz 闪烁脉冲
  - 整条横向 ±1.5px 高频抖动
- **D 操作命令词紧贴指针下方**（贴在玩家视线焦点）：
  - 左红区："❗ 按住"（浅金）/ 右红区："❗ 松手"（浅粉）
  - 黑底胶囊 + 金/粉描边，24×24 紧贴指针下沿 14px
- 删除原屏幕中央"快收线/快放线"文字（远离视线焦点，无效）

所有反馈集中在 ▲ 指针周围 ~30×30 像素区域，玩家不需要移视线。

### A9. HP=0 死循环 hotfix-x5
**现象**：鱼血归零后仍游动+回血，无法上岸。
**根因**：hotfix-y2（光弹飞行 300ms 命中扣血）引入新 bug：
1. tick 触发发射致命光弹（damage=12，鱼 HP=10）
2. **300ms 飞行期间** `fishHP > 0` 仍成立 → 玩家放松控压时**回血逻辑还在跑**，HP 10 → 10.3
3. 光弹命中：`Math.round(min(12, 10.3)) = 10`，扣完 HP = **0.3 浮点残血**
4. 0.3 永远 > 0 → `<= 0` 锁存判定不触发 → 回血继续 → **永远扣不死**

**修复**（双保险）：
- 回血逻辑加 `!hasFlyingProjectile` 条件 → 光弹飞行期间禁用回血（根因）
- 回血逻辑加 `fishHP >= 1.0` 守卫
- 命中后残血 < 1.0 钳到 0（兜底浮点累计误差）

### A10. 放大镜"放弃"胶囊（删除"重新抛竿"+ 小花瓣方案）
**演化**：
1. 老板初提：双花瓣（蓝刺鱼+红收回） → 实施
2. 验收后老板裁定简化：**删刺鱼**（保留键盘空格/点鱼），保留单红色"放弃"胶囊
3. 形态：圆角胶囊 + 红底（`#E74C3C`）+ 米白文字"放弃"+ 顶部线性高光
4. **位置：浮漂正下方**（不是放大镜下方）
5. **越界保护**：浮漂贴近画幅底部 → 翻到上方；左右 clamp 贴边

### A11. 放大镜玻璃高光删除
**现象**：放大镜左上角弧形白色高光被误识为"读条/进度条"。
**修复**：删除 `_drawFrame` 内的弧形高光绘制 + 删除 `HILIGHT_COLOR` 常量。镜片仅保留：内暗角、金色边框、深棕内描边。

## 3. NPC / 任务体验

### A12. q001 奇力鱼权重 ×2（BL-001 方案 E）
**反馈**：玩家接 q001 后体感"很难钓到奇力鱼"，期望 17 次抛竿才能完成 3 条任务。
**数据现状**：奇力鱼 weight 35，抽中率 30.4%；3条÷30%÷60%成功率 ≈ 17 次抛竿。
**方案 E（数据 + 叙事）**：
- `fish-pool.js` 新增 `_isQ001Active()`，q001 active 时 `qiliyu` weight × 2 → **抽中率 30.4% → 46.7%**
- 任务完成后自动恢复（`getStatus` 返回 `'completed'`）
- `dialogues.js` `chief_quest_offer` 在阿明回应"奇力鱼？就是潭里那种银白色的小鱼对吧"之后插入：
  > 阿土伯：**这季节潭里奇力鱼最多了，你抛几竿就能撞上一条。**

数据 + 叙事一致，玩家会觉得"村长说的没错"。

### A13. 任务面板显示 ID 修复
**现象**：q001 接取后任务面板顶部显示 `q001_first_fish` 而非"潭边的早晨"。
**根因**：`window.QUESTS?.[q.id]` 永远 undefined（QUESTS 是 ES module import，没挂 window）→ 一直走 fallback `{ name: q.id }` 显示 ID。
**修复**：用 import 进来的 `QUESTS` + 兼容 q001 用 `name` 字段、q002/q003 用 `title` 字段：
```js
const tpl = QUESTS[q.id] || {};
const displayName = tpl.name || tpl.title || q.id;
```
三个任务现在都能正确显示中文名（潭边的早晨 / 潭中百味 / 林师傅的考验）。

## 4. 村庄 UI 增强

### A15. 小地图右下 3 图标快捷按钮（重头戏）
**需求**：背包/图鉴/任务三个面板的打开方式增加图标按钮，紧贴小地图圆框右下，鼠标悬停显示带快捷键的 tooltip。

**实现**：
- 新方法 `_renderMinimapBtns(ctx, centerX, centerY, radius)`，3 个按钮沿圆弧排列：
  - 🎒 背包 (B) → 20°
  - 📖 鱼图鉴 (T) → 45°
  - 📋 任务面板 (Q) → 70°
- `btnDist = radius + btnR + 4`（贴边 4px）
- 按钮 32×32 圆形：黑底（默认）/ 金底（hover）/ 黑描边（默认）/ 金描边（hover）
- emoji 图标居中（🎒/📖/📋）
- **tooltip**：14px 米白文字 + 黑底圆角 + 金描边，`"打开XX (B/T/Q)"`，按钮上方；越界翻下方，水平 clamp
- 圆形精确命中（dx²+dy² ≤ r²）
- 同优先级 noPanel 守卫（与游戏指南 HUD 一致）：对话/任意面板打开时不响应
- 点击 = 等价对应键盘快捷键 + `AudioSystem.playMenuOpen()`

**几何调试历程**：
- v1 角度差 17°/btnDist 101 → 按钮重叠
- v2 拉到 btnDist 111 + 角度差 25° → 不重叠但太"飘"
- v3 改回 btnDist 101 + 角度差 30° → 第三按钮压 S 方向标
- **v4 最终**：btnDist 101 + 角度 20°/45°/70°（角差 25°，避开 S 标 90°）→ 完美

## 7. 系统性问题（已建档，待 D15+ 处理）

### BL-002：项目所有 `<img>` 资源没带版本号
**风险**：每次替换贴图，玩家不清缓存就看不到新图。
**影响范围**：约 14 处 `img.src`，含 NPC/招牌/鱼图等。
**建议方案**：
- 短期：每个资源替换时手动给该 src 加 `?v=YYYYMMDD`（fishing-bg 已示范）
- 长期：建一个 `js/render/asset-versions.js` 统一常量表，所有 loader 从中读取版本号

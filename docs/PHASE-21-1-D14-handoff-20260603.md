## 1. 落地范围

### 1.1 §1 鱼群随机化（每局开场 roll，局内锁定）

- 3 群 left/center/right，centerX/Y/fishCount/wanderArea 全部随机
- fishCount 每群独立 randInt(3,8)，wanderArea.w 140-240 / .h 90-150
- 群间 AABB 不交（GAP_X=40 / GAP_Y=30），最多 20 次重试，失败回退 FALLBACK_LAYOUT
- 体型每条独立伯努利 roll：`{small:.50, medium:.35, large:.15}`，删稀疏档保底
- hover 浮窗：删金色呼吸描边、"鱼群密度" → "资源丰富度"、视觉档 ≤3/4-6/≥7

### 1.2 §2 受惊系统 + §扩展三段曲线

- `applyScare(bobX, bobY)`：浮漂落水触发，R_SCARE=120px 内整群进 flee
- **三段曲线**（每条独立 roll）：
  - **panic 期 350-450ms**：方向每 100-150ms reroll，最多 2-5 次后锁定；速度 75-110 px/s；朝四面八方乱跳 → 慌张乱窜感
  - **escape 期 250-900ms**：方向锁定"远离浮漂 ±30°"，速度 50-110 px/s
  - **decay 期 → fleeTimer 末**：easeOutCubic 衰减到 4 px/s 接近停滞
- **逃跑 y 锁定屏幕下半部**（y ≥ 360），撞墙翻转 fleeDirY
- **笨鱼豁免**：每条 10% 概率不进 flee（呆呆继续 wander）；已 flee 的笨鱼二次受惊会"反应过来"切回 wander
- **总持续 2.5-5s**，fleeTimer 到 → 切 returning 30 px/s 慢游回群中心
- **returning 期不被 wanderArea 边界 clamp**（防"瞬移回来变新鱼"）

### 1.3 §3 前戏四态 BobberApproachFSM（替代旧 fishShadow）

- Waiting 期 1.5-4s 后启动 searching → 从最近 wander 群挑 1 条鱼 approach 浮漂
- approach 45 px/s 朝 bobX/bobY，抵达 ±15px → 切 biting + roll 咬钩类型
- **概率分布**（spec §3.4 锁死）：试探 35% / 偷吃 30% / 真吃 25% / 黑漂 10%
- **浮漂前戏晃动**（每帧 100ms，dx 帧表）：
  - 试探：`[-2, 0, +2, 0]` × 400ms
  - 偷吃：`[-3, 0, +3, 0, ...]` × 800ms
  - 真吃：`[-4, 0, +4, 0, ...]` × 1200ms → 交棒 D5 走完整 shake→sink
  - 黑漂：不晃 800ms → 交棒 D5 短路（跳 shake 直入 sink）
- Waiting 期空格/鼠标左键 = 提竿：
  - 试探/偷吃晃动期 → 空竿 toast`🪝 空竿了…`（降档 TD-D14-6 待实现）
  - 真吃晃动期 → 温柔早提（空竿，不降档）
- 派出去的鱼若 approach/biting 期被受惊 → 整群进 flee，前戏 FSM 重启

### 1.4 §4 删除旧 fishShadow

- 删 `_updateWaiting` L940-961 整段"左右随机生成 fishShadow + 朝浮漂游 + 概率 _startBite"
- 删 `_renderFishShadow()` 函数 + `_render()` 中渲染守卫
- 删 5 处字段重置（L217 / L920 / L933 / L1587 / L1596）
- `CONFIG.waiting.fishSpawnDelay` / `CONFIG.fish.travelSpeed` 标 `@deprecated D14`
- 全文 grep `fishShadow` 仅在注释中出现

### 1.5 D5 接口扩展（4 行）

- `D5BiteFeedback.onBiteStart(level, opts)` 内 `opts.isLuckyBail===true` 时把 `biteWindowElapsed` 推到 `w.shakeEnd`，下一帧 `_updateZone` 切 perfect，shake→sink 边沿条件触发 onSinkStart
- 这就是黑漂"跳 shake 直入 sink"的实现，符合 spec §3.6 "≤5 行 if 分支"约束

## 2. 实施期 hotfix 实录（10 轮老板拍板调优）

| Hotfix | 内容 | 触发 |
|---|---|---|
| a | `_startBite` 缺 `_selectFish()` 导致 `currentFish=null` → `_initPlayingState` NPE | 提竿后报错 |

| e | panic 期太短（200ms） + 逃距同质化 → 每条独立时长 + 拉大 dash 范围 | 不够散 |
| f | panic 期再加长 350-450ms + 锁屏下半部（撞墙反弹） | 美术约束 |
| g | panic 速度减半（150-220 → 75-110） | 太快 |
| h | reroll 间隔 60-80ms + 限制次数 2-5 | 节奏感 |
| i | reroll 再降到 100-150ms | 节奏感 |
| j | escape/decay 速度减半（逃距压到 50-125px） | 距离收敛 |
| k | 10% 笨鱼豁免（呆鱼不逃） | 真实塘里的鱼 |

## 2. TILE 枚举

| ID | 名称       | 视觉含义       | 是否可走 (WALKABLE) |
|---:|-----------|--------------|:------------------:|
|  0 | `GRASS`   | 草地         | ✅ 可走 |
|  1 | `DIRT`    | 泥土路       | ✅ 可走 |
|  2 | `STONE`   | 石砖广场     | ✅ 可走 |
|  3 | `WOOD`    | 栈桥（钓鱼触发）| ✅ 可走 |
|  4 | `SHALLOW` | 浅水（保留未用）| ❌ 阻挡 |
|  5 | `DEEP`    | 深水（湖面） | ❌ 阻挡 |
|  6 | `ROOF`    | 建筑屋顶     | ❌ 阻挡 |
|  7 | `WALL`    | 建筑墙身     | ❌ 阻挡 |
|  8 | `TREE`    | 树木 / 边界  | ❌ 阻挡 |
|  9 | `FISHING` | 水中钓点圆圈 | ❌ 阻挡（PHASE 13-6 改）|

**判定规则（铁律）**：
```
WALKABLE = { GRASS, DIRT, STONE, WOOD }
任何不在此集合的 tile 都是"空气墙"，玩家无法移入。
越界（tx<0 或 tx≥COLS 或 ty<0 或 ty≥ROWS）也按 TREE 处理 → 阻挡。
```

## 4. 建筑物理框（不参与碰撞，仅供"靠近交互"判定）

`BUILDINGS`（`js/village-scene.js:139`）记录 4 栋房子的渲染锚点（左上角像素坐标）。  
**它们不直接产生空气墙**——空气墙完全由上表 ROOF + WALL 的 tile 网格决定。BUILDINGS 仅用于：① 渲染建筑 sprite ② 玩家靠近时弹出"按 E 进入"提示。

## 6. 玩家碰撞判定（运行时）

源码：`js/village-scene.js:1881-1883` + `js/village-scene.js:1989-2010`

### 6.1 单点判定
```js
_canMoveTo(tx, ty) → WALKABLE.has(tile_at(tx, ty))
```
越界返回 TREE（即不可走）。

### 6.2 玩家 hitbox（半身）
```
hbCx = nextPx                 // 像素中心 x
hbCy = nextPy + T * 3/8       // 像素中心 y（脚下 12px 偏移）
hbHalf = 8 px                 // 半宽 = 8px → hitbox 实际宽 16px、高 16px
```

### 6.3 X / Y 分轴检测（避免"贴墙抖动"）
```
横向移动可行：
  tx_x = floor(hbCx / T)
  ty_min = floor((hbCy - 8) / T)
  ty_max = floor((hbCy + 8 - 1) / T)
  canX = canMoveTo(tx_x, ty_min) && canMoveTo(tx_x, ty_max)

纵向移动可行：
  tx_min = floor((hbCx - 8) / T)
  tx_max = floor((hbCx + 8 - 1) / T)
  ty_y = floor(hbCy / T)
  canY = canMoveTo(tx_min, ty_y) && canMoveTo(tx_max, ty_y)

if (canX) px = nextPx;
if (canY) py = nextPy;
```

### 6.4 钓鱼触发
玩家所在 tile 类型 == `WOOD`（栈桥）→ `onFishingSpot = true`，按 F 开钓。  
`FISHING` tile **不可走**（仅作视觉钓点圆圈），玩家踩不到。

## 8. 修改 SOP（给 Knot）

### 8.1 想"加一道空气墙"
1. 在第 3 节填充表里新增一行，决定 TILE 类型（推荐 `TREE` 或 `WALL`）。
2. 算好 `(x0, y0, x1, y1)`（闭区间，T=32 网格）。
3. 同步到 `js/village-scene.js::_generateMap()` —— 找到合适顺序（注意"后写覆盖前写"）插入一句 `fillRect(TILE.XXX, x0, y0, x1, y1);`。
4. 同步 `scripts/dump-village-collision.py::generate_map()`（同样的一行 Python 写法）。
5. 重跑脚本生成新 PNG：`python3 scripts/dump-village-collision.py`。
6. 更新本文 ASCII 总览（可选）。

### 8.2 想"打开一道现有的墙"
1. 找出阻挡来源——查第 3 节，定位是哪步 `fillRect` 写进去的。
2. 三种处理方式（任选）：
   - **缩范围**：缩短那一步的 (x0..x1) / (y0..y1)。
   - **后置覆盖**：在那一步之后再用 `fillRect(TILE.GRASS/DIRT, ...)` 覆盖回可走。
   - **改 TILE 类型**：把这块 tile 整体换成 WALKABLE 集合的成员。
3. 同步 JS + Python 脚本。

### 8.3 想"让某个 tile 类型从阻挡变可走（或反过来）"
**只需改一行**：`js/village-scene.js:133` 的 `WALKABLE` 集合。  
但要警觉视觉一致性——例如让 DEEP 可走，玩家会走进湖里。

### 8.4 想"调整玩家 hitbox 大小"
改 `js/village-scene.js:1997` 的 `hbHalf`（默认 8 → 半宽 8px = 一格的 1/4）。改大会更难钻空隙、改小会更灵活。

### 8.5 想"调钓鱼触发区"
改 `js/village-scene.js:2038`：
```js
this.onFishingSpot = (this._getTileAt(this.player.tx, this.player.ty) === TILE.WOOD);
```
若想让"任意水边"都能钓鱼，比如把 `=== TILE.WOOD` 改成 `=== TILE.WOOD || ...`。注意 `FISHING` tile 不可走，玩家脚下永远不会是 FISHING。

## 0. 设计总纲（Design Pillars）

D5 视觉规范的三条北极星：

1. **辨识高于装饰**：三档（轻/中/猛）信号差异度 > 美感统一度。玩家在 0.3s 内必须分辨。
2. **浮漂本体不动，外周做戏**：浮漂 11×24 像素栅格不重画，所有差异化通过 ① 抖动模式 ② 放大镜边框 ③ 镜头微震 ④ 外周 FX（水纹/水花）四层叠加完成。
3. **休闲不苦情**：错过有"诶可惜"的失落感，不要"完蛋了"的挫败感。所有警示色短促即收，不长时间停留。

**复用色板**（与双轴搏斗 UI 共用 10 色，新增 4 色见 §8.4）：

## 1. 三档抖动动画规格 ★★★（最重要）

### 1.1 设计哲学：三档差异化的"三轴策略"

为保证玩家"看得出来"，三档抖动必须在**三个独立轴**上同时拉开差距：

| 差异化轴 | 轻档 | 中档 | 猛档 |
|---|---|---|---|
| **A 轴：垂直位移幅度** | 3px | 6px | 12px |
| **B 轴：节奏密度** | 2~3 次/秒（疏） | 4~5 次/秒（中） | 1 次大动作（炸） |
| **C 轴：横向位移** | 0px | 0~1px | 4~6px |

**记忆口诀**：轻=点头、中=连续啄、猛=黑漂横拉。

### 1.2 帧表（按 12fps 像素动画基准，1 帧=83ms）

#### 🟢 轻档 Light：`float-shake-light-8f.png`

- **总时长**：667ms（8 帧 @ 12fps）

| 帧 | Δy | Δx | 说明 |
|---|---|---|---|
| 0 | 0 | 0 | 静帧（锚点） |
| 1 | +2 | 0 | 第一次小点头 |
| 2 | +1 | 0 | 回弹一半 |
| 3 | 0 | 0 | 短暂归位 |
| 4 | +3 | 0 | 第二次稍大点头 |
| 5 | +2 | 0 | 缓回 |
| 6 | +1 | 0 | 进入稳定下沉 |
| 7 | +1 | 0 | 收 |

- **缓动**：帧 1→2、4→5 用"停 1 帧"模拟 ease-out
- **循环规则**：单次播放 8 帧后无缝接 BiteWindow 持续态（每 250ms 多 1px 缓沉）

#### 🟡 中档 Medium：`float-shake-medium-10f.png`

- **总时长**：833ms（10 帧 @ 12fps）

| 帧 | Δy | Δx | 说明 |
|---|---|---|---|
| 0 | 0 | 0 | 静帧 |
| 1 | +4 | 0 | 啄①下 |
| 2 | +1 | 0 | 啄①回 |
| 3 | +5 | 0 | 啄②下 |
| 4 | +2 | 0 | 啄②回 |
| 5 | +2 | 0 | **小停顿（关键差异点）** |
| 6 | +2 | 0 | **小停顿持续** |
| 7 | +6 | +1 | 啄③下 |
| 8 | +4 | 0 | 缓沉 |
| 9 | +5 | 0 | 收 |

- **节奏特征**：「啄-啄-停-啄」4 拍节奏，第 5/6 帧的"停"是中档灵魂

#### 🔴 猛档 Heavy：`float-shake-heavy-6f.png`

- **总时长**：500ms（6 帧 @ 12fps），**前 2 帧可拉到 16fps 加速突袭感**

| 帧 | Δy | Δx | 说明 |
|---|---|---|---|
| 0 | 0 | 0 | 静帧 |
| 1 | +5 | -3 | **黑漂启动+横向左拽** |
| 2 | +10 | -5 | 急沉到底 |
| 3 | +12 | +4 | **横向反向拉拽** |
| 4 | +12 | -2 | 继续拉 |
| 5 | +12 | 0 | 锁定深沉位 |

- **节奏特征**：无回弹，一次性炸下去，横向左右反复拉
- **持续态**：6 帧后浮漂保持深沉位 + 1Hz 横向 ±2px 微抖直到玩家提竿

### 1.3 与浮漂栅格 11×24 的兼容性

- **画布预留**：3 张 sprite sheet 单帧画布统一 **24×40 像素**
- **锚点**：浮漂底端中心点 (x=12, y=32)
- **像素对齐**：所有位移整数像素（无亚像素）
- **缩放**：动画在原始 1× 尺寸绘制，运行时统一应用 ×0.8 缩放

### 1.4 信号差异度的"四重保险"

| 保险层 | 轻档 | 中档 | 猛档 |
|---|---|---|---|
| ① 浮漂抖动 | 小幅点头 | 啄停啄节奏 | 黑漂横拉 |
| ② 放大镜边框 | 无变化 | 无变化 | 边框泛红 |
| ③ 镜头微震 | 无 | 无 | 60ms 微震 |
| ④ 外周水纹 | 小波 | 中波 | 大水花溅起 |

## 2. 放大镜边框染色规范

### 2.1 猛档"边框泛红" `magnifier-frame-tint: heavy-red`

- **基色**：`#C8412B`
- **染色范围**：放大镜外圈 4px 木质边框
- **混合模式**：multiply
- **透明度曲线**：

| 时间点 | α |
|---|---|
| 0ms | 0% |
| 80ms | 70% |
| 200ms | 70% |
| 400ms | 50% |
| 500ms | 0% |

- **触发时机**：与猛档抖动帧 0 对齐（`onBiteStart` + `intensity=heavy`）

### 2.2 LATE 区段警示泛红（与猛档区分）

- **基色**：`#E8553D`（**新增色**：偏橙红）
- **行为模式**：**呼吸式持续闪烁**
  - 频率：2Hz（500ms 一个完整呼吸）
  - 透明度曲线：0% → 60% → 0%（正弦波）
  - 持续到玩家提竿或离开 LATE 区段

| 维度 | 猛档边框泛红 | LATE 警示泛红 |
|---|---|---|
| 色相 | 纯警示红 `#C8412B` | 偏橙红 `#E8553D` |
| 持续 | 一次性 500ms | 持续呼吸到提竿 |
| 节奏 | 急亮急退 | 2Hz 平稳呼吸 |
| 配套 | 镜头微震+水花 | 仅边框 |

**冗余保险**：LATE 警示同时在浮漂下方加 ⚠ 像素小标（8×8，`#E8553D`）。

## 3. 镜头微震参数（仅猛档）

- **挂载点**：`cameraMicroShake`
- **触发时机**：`onBiteStart`（**不挂在** `onBiteCharge`，避免过早抖屏干扰决策）
- **波形**：**伪随机抖**
  - 每 10ms 在 [-2, +2] 像素范围内随机取一个 (Δx, Δy) 偏移
  - 60ms 共 6 次随机偏移，最后归零
- **像素对齐**：随机值取整
- **影响范围**：整个游戏画布（包括 UI 层）
- **频率限制**：同一咬钩窗口内只触发 1 次

## 4. 漏提演出 800ms 分镜

### 4.1 时间轴分镜

| 时间段 | 画面元素 | 描述 |
|---|---|---|
| 0~200ms | 浮漂 | 快速回弹归位（4 帧 @ 12fps） |
| 100~600ms | 涟漪扩散 | 复用 1.4s 涟漪 + 参数化改造 |
| 200~700ms | 鱼影远离 | 半透明色块向画面下方游走 |
| 400~800ms | "可惜……"文案 | 浮漂上方淡入淡出 |

### 4.2 涟漪扩散：**复用 + 改造**（不新建动画）

- **复用源**：现有 1.4s 周期 3 层错峰扩散椭圆
- **改造点**（运行时参数化）：
  - 触发位置：浮漂当前位置
  - 扩散速度：×1.5（800ms 完成原本 1.4s 的扩散）
  - 透明度起始：60% → 0%
  - 仅触发 1 个椭圆

### 4.3 鱼影远离

- **复用** `fish_shadow_3layers.png` 远景层（24×12 像素，`#1A3A4A`，25%α）
- **运动路径**：从浮漂下方 16px 处沿 30° 斜向下，500ms 内位移 64px
- **透明衰减**：25%α → 0%α
- **理由**：远景层是模糊剪影，看不出鱼种（不剧透红线）

### 4.4 "可惜……"文案

- **字号**：12px（比 PERFECT 字 16px 小一档）
- **颜色**：`#A89888`（**新增色**：低饱和米灰）
- **描边**：1px `#1A1A1A`
- **位置**：浮漂上方 24px 居中
- **淡入淡出**：

| 时间 | α |
|---|---|
| 0ms | 0% |
| 150ms | 100% |
| 350ms | 100% |
| 500ms | 0% |

- **关键设计**：**不加金光、不加震动、不加感叹号**

## 9. 5 人小测视觉验收 DoD

### 9.1 玩家应该能"看出"的 3 个核心视觉差异（必过项）

| # | 验收项 | 通过标准 |
|---|---|---|
| 1 | **三档力度区分** | 5 人中 ≥4 人对 9 次咬钩（每档 3 次）正确分类 ≥80% |
| 2 | **猛档与 LATE 警示区分** | 5 人中 ≥4 人能区分"边框红了一下"vs"边框一直在闪" |
| 3 | **PERFECT 三档区分** | 5 人中 ≥4 人答对"稳/妙/完美"哪一档 ≥80% |

### 9.2 玩家不应该"看混"的 2 个反例（红线项）

| # | 反例 | 排查标准 |
|---|---|---|
| 1 | **四叶草微光被识别为"系统提示"** | ≤1 人主动提到"那个绿色的小标是不是给我开挂" |
| 2 | **"可惜……"被解读为"扣分/惩罚"** | 0 人产生挫败感表述 |

### 9.3 兜底验收

- **像素风一致性**：所有新增资产无亚像素、无抗锯齿羽化边
- **调色板合规率**：新增资产用色 100% 落在 8 色 + 4 新增色范围内
- **浮漂栅格未被破坏**：抖动 sprite sheet 中浮漂本体仍为 11×24 严格栅格

## 10. 关键设计决策溯源

| 决策 | 主美理由 |
|---|---|
| 三档差异化用「三轴策略+四重保险」 | 单维差异化（如只拉幅度）易被色弱/眼花玩家误读，三轴叠加+四重冗余兜底 |
| 浮漂本体不重画 | 11×24 栅格是 v3.0 已锁定资产，重画 = 整套钓鱼 UI 联调风险 |
| 中档 PERFECT 拍「妙！」 | 语义梯度 + 单字易读 + 台湾文化语境契合 |
| 四叶草绿+金双色 | 绿=Lucky 文化通感，金=惊喜光泽，避免与水面反光（白色）混淆 |
| 漏提鱼影复用远景层 | 模糊剪影看不出鱼种，规避"BiteWindow 不剧透"红线 |
| 涟漪运行时参数化复用 | 不出新资产，节省美术工作量 + 风格一致 |
| 镜头微震挂 onBiteStart | onBiteCharge 期间玩家在判断要不要提竿，过早抖屏干扰决策 |
| LATE 警示用偏橙红呼吸式 | 与猛档"纯红一闪"形成色相+节奏双重区分，避免混淆 |

## 0. 设计总纲（Design Pillars）

D5 视觉规范的三条北极星：

1. **辨识高于装饰**：三档（轻/中/猛）信号差异度 > 美感统一度。玩家在 0.3s 内必须分辨。
2. **浮漂本体不动，外周做戏**：浮漂 11×24 程序化绘制（v3.0 `_drawPixelBob`）不重写，所有差异化通过 ① **运行时偏移帧表** ② 放大镜边框 ③ 镜头微震 ④ 外周 FX（水纹/水花）四层叠加完成。
3. **休闲不苦情**：错过有"诶可惜"的失落感，不要"完蛋了"的挫败感。所有警示色短促即收，不长时间停留。

## 1. 浮漂三档抖动规格 ★★★（核心）

### §1.0 实现路线（A+C 混合方案）

浮漂三档抖动**不出 PNG sprite sheet**。v3.0 浮漂为程序化绘制（`_drawPixelBob(cx, cy, clipBottom)`），主程在该函数调用前对入参 `cx/cy` 应用 `(Δx, Δy)` 偏移即可达成抖动效果。本节定义偏移量帧表与节奏曲线，由前端实现为运行时参数。

### §1.1 调用契约

```js
// 主程对接示例（每帧 12fps）
const { dx, dy } = BiteShakeFrame.sample(level, frameIdx);
this._drawPixelBob(bobX + dx, bobY + dy, clipBottom);
```

- `level`: `'light' | 'medium' | 'heavy'`
- `frameIdx`: 当前帧索引，超出帧数后归 0（轻档循环、中档循环、猛档单次播放至结束态保持）
- `dx/dy`: 整数像素，已含 `ctx.scale(0.8)` 外层缩放补偿（即帧表数值为屏幕像素，主程无需再换算）

### §1.2 三档抖动差异化方法论

为保证玩家"看得出来"，三档抖动必须在**三个独立轴**上同时拉开差距：

| 差异化轴 | 轻档 | 中档 | 猛档 |
|---|---|---|---|
| **A 轴：垂直位移幅度** | 3px | 6px | 12px |
| **B 轴：节奏密度** | 2~3 次/秒（疏） | 4~5 次/秒（中） | 1 次大动作（炸） |
| **C 轴：横向位移** | 0px | 0~1px | 4~6px |

**记忆口诀**：轻=点头、中=连续啄、猛=黑漂横拉。

### §1.3 三档偏移帧表（按 12fps，1 帧 = 83ms）

| 档 | 帧数/总时长 | 节奏特征 | (Δx, Δy) 帧序列 |
|---|---|---|---|
| 🟢 轻 | 8 帧 / 667ms | 点头-回弹-点头-缓沉 | `(0,0)(0,2)(0,1)(0,0)(0,3)(0,2)(0,1)(0,1)` |
| 🟡 中 | 10 帧 / 833ms | 啄-啄-**停**-啄（停顿是灵魂） | `(0,0)(0,4)(0,1)(0,5)(0,2)(0,2)(0,2)(1,6)(0,4)(0,5)` |
| 🔴 猛 | 6 帧 / 500ms | 黑漂横拉，无回弹 | `(0,0)(-3,5)(-5,10)(4,12)(-2,12)(0,12)` |

> 锚点为浮漂底端中心（与 `_drawPixelBob` 内部坐标系一致），所有偏移为整数像素，**不做亚像素插值**（保持像素风硬边）。
> 持续态（帧序列结束后）：轻/中循环回帧 0；猛档保持帧 5 + 1Hz 横向 ±2px 微抖直到玩家提竿。

### §1.4 程序化绘制运行时参数传递规范

1. **接入点**：仅 `_drawPixelBob` 调用前一行加偏移量，函数本体禁止修改（保留 v3.0 hotfix h/i/j 的稳定性）。
2. **clipBottom 不受影响**：抖动仅平移 `cx/cy`，浮漂落水裁剪逻辑（row 12-23 被裁）继续生效。
3. **三轴差异**（A 垂直/B 节奏/C 横向）由 §1.3 帧表数值天然承载，无需额外开关。
4. **四重保险**中浮漂抖动一项即为本节，其余 3 重（放大镜染色 / 镜头微震 / 水花）维持原方案。
5. **降级**：若 `BiteShakeFrame` 模块未加载，主程 fallback 为 `(0, 0)`（即不抖），不应崩溃。

### §1.5 信号差异度的"四重保险"

| 保险层 | 轻档 | 中档 | 猛档 |
|---|---|---|---|
| ① 浮漂抖动 | 小幅点头（程序化） | 啄停啄节奏（程序化） | 黑漂横拉（程序化） |
| ② 放大镜边框 | 无变化 | 无变化 | 边框泛红 |
| ③ 镜头微震 | 无 | 无 | 60ms 微震 |
| ④ 外周水纹 | 小波 | 中波 | 大水花溅起 |

## 2. 放大镜边框染色规范

### 2.1 猛档"边框泛红" `magnifier-frame-tint: heavy-red`

- **基色**：`#C8412B`（待与 v3.0 描边色 `#1A1A2E` 协调测试，本色为主美外层叠加色，不冲突）
- **染色范围**：放大镜外圈 4px 木质边框
- **混合模式**：multiply
- **透明度曲线**：

| 时间点 | α |
|---|---|
| 0ms | 0% |
| 80ms | 70% |
| 200ms | 70% |
| 400ms | 50% |
| 500ms | 0% |

- **触发时机**：与猛档抖动帧 0 对齐（`onBiteStart` + `intensity=heavy`）

### 2.2 LATE 区段警示泛红（与猛档区分）

- **基色**：`#E8553D`（**新增色**：偏橙红）
- **行为模式**：**呼吸式持续闪烁**
  - 频率：2Hz（500ms 一个完整呼吸）
  - 透明度曲线：0% → 60% → 0%（正弦波）
  - 持续到玩家提竿或离开 LATE 区段

| 维度 | 猛档边框泛红 | LATE 警示泛红 |
|---|---|---|
| 色相 | 纯警示红 `#C8412B` | 偏橙红 `#E8553D` |
| 持续 | 一次性 500ms | 持续呼吸到提竿 |
| 节奏 | 急亮急退 | 2Hz 平稳呼吸 |
| 配套 | 镜头微震+水花 | 仅边框 |

**冗余保险**：LATE 警示同时在浮漂下方加 ⚠ 像素小标（8×8，`#E8553D`）。

## 3. 镜头微震参数（仅猛档）

- **挂载点**：`cameraMicroShake`
- **触发时机**：`onBiteStart`（**不挂在** `onBiteCharge`，避免过早抖屏干扰决策）
- **波形**：**伪随机抖**
  - 每 10ms 在 [-2, +2] 像素范围内随机取一个 (Δx, Δy) 偏移
  - 60ms 共 6 次随机偏移，最后归零
- **像素对齐**：随机值取整
- **影响范围**：整个游戏画布（包括 UI 层）
- **频率限制**：同一咬钩窗口内只触发 1 次

## 4. 漏提演出 800ms 分镜

### 4.1 时间轴分镜

| 时间段 | 画面元素 | 描述 |
|---|---|---|
| 0~200ms | 浮漂 | 快速回弹归位（4 帧 @ 12fps） |
| 100~600ms | 涟漪扩散 | 复用 1.4s 涟漪 + 参数化改造 |
| 200~700ms | 鱼影远离 | 半透明色块向画面下方游走 |
| 400~800ms | "可惜……"文案 | 浮漂上方淡入淡出 |

### 4.2 涟漪扩散：**复用 + 改造**（不新建动画）

- **复用源**：现有 1.4s 周期 3 层错峰扩散椭圆
- **改造点**（运行时参数化）：
  - 触发位置：浮漂当前位置
  - 扩散速度：×1.5（800ms 完成原本 1.4s 的扩散）
  - 透明度起始：60% → 0%
  - 仅触发 1 个椭圆

### 4.3 鱼影远离

- **复用** `fish_shadow_3layers.png` 远景层（24×12 像素，`#1A3A4A`，25%α）
- **运动路径**：从浮漂下方 16px 处沿 30° 斜向下，500ms 内位移 64px
- **透明衰减**：25%α → 0%α
- **理由**：远景层是模糊剪影，看不出鱼种（不剧透红线）

### 4.4 "可惜……"文案

- **字号**：12px（比 PERFECT 字 16px 小一档）
- **颜色**：**`#C8B89A`**（复用 v3.0 浮漂白身暗部 `WHT_DK`，原拟用 `#A89888` 因孤立色作废）
- **描边**：1px `#1A1A2E`
- **位置**：浮漂上方 24px 居中
- **淡入淡出**：

| 时间 | α |
|---|---|
| 0ms | 0% |
| 150ms | 100% |
| 350ms | 100% |
| 500ms | 0% |

- **关键设计**：**不加金光、不加震动、不加感叹号**

## 10. 关键设计决策溯源（v1.1 增补）

| 决策 | 主美理由 |

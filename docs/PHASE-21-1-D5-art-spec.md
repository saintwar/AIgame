## 0. 设计总纲（Design Pillars）

D5 视觉规范的三条北极星：

1. **辨识高于装饰**：三档（轻/中/猛）信号差异度 > 美感统一度。玩家在 0.3s 内必须分辨。
2. **浮漂本体不动，外周做戏**：浮漂 11×24 程序化绘制（v3.0 `_drawPixelBob`）不重写，所有差异化通过 ① 运行时偏移帧表 ② 放大镜边框 ③ 镜头微震 ④ 外周 FX（水纹/水花）四层叠加完成。
3. **休闲不苦情**：错过有"诶可惜"的失落感，不要"完蛋了"的挫败感。所有警示色短促即收，不长时间停留。

### §0.3 历史更正

**v1.0 → v1.1**（2026-06-01 早）：误标 `#1A1A1A`、`#C8412B` 红锥色、缺失黄环段，全部作废。

**v1.1 → v1.2**（2026-06-01 晚，全程序化路线）：

v1.1 §8 资产清单将 ④⑤⑥⑧⑨⑩⑪ 共 7 项归为「AI 生图」是工程视角误判，忽略三项硬约束：

1. **像素硬边底线**：D5 与双轴搏斗 / v3.0 浮漂同框，AI 生图反锯齿/半透明边缘破坏整工程像素硬边一致性
2. **色板合规**：12 色调色板必须严格命中，AI 生图色彩漂移率高
3. **与 v3.0 同框统一**：浮漂、PERFECT 字、放大镜染色都挂在 `fishing-scene.js` 同一 Canvas 上下文

**结论**：除「鱼影远景」复用既有 sprite 外，其余可绘制资产**全部程序化**。**0 张 AI 生图、0 token 消耗**。

**经验沉淀**：像素游戏的小尺寸视觉资产（≤32×32、几何规整、色板 ≤16 色），程序化绘制的边际成本通常低于 AI 生图——后者在像素硬边、色板合规、缩放对齐三项硬约束下需要重度后处理。规范评估生图前，必须先评估「能否 30 行代码画出来」。

## 1. 浮漂三档抖动规格 ★★★（核心）

### §1.0 实现路线

浮漂三档抖动**不出 PNG**。v3.0 浮漂为程序化绘制（`_drawPixelBob(cx, cy, clipBottom)`），主程在该函数调用前对入参 `cx/cy` 应用 `(Δx, Δy)` 偏移即可。

### §1.1 调用契约

```js
const { dx, dy } = BiteShakeFrame.sample(level, frameIdx);
this._drawPixelBob(bobX + dx, bobY + dy, clipBottom);
```

- `level`: `'light' | 'medium' | 'heavy'`
- `frameIdx`: 当前帧索引，超出后归 0
- `dx/dy`: 整数像素，已含 `ctx.scale(0.8)` 外层缩放补偿

### §1.2 三档差异化方法论（三轴策略）

| 差异化轴 | 轻档 | 中档 | 猛档 |
|---|---|---|---|
| **A 垂直位移** | 3px | 6px | 12px |
| **B 节奏密度** | 2~3 次/秒（疏） | 4~5 次/秒（中） | 1 次大动作（炸） |
| **C 横向位移** | 0px | 0~1px | 4~6px |

记忆口诀：轻=点头、中=连续啄、猛=黑漂横拉。

### §1.3 三档偏移帧表（12fps）

| 档 | 帧数/总时长 | 节奏特征 | (Δx, Δy) 帧序列 |
|---|---|---|---|
| 🟢 轻 | 8 帧 / 667ms | 点头-回弹 | `(0,0)(0,2)(0,1)(0,0)(0,3)(0,2)(0,1)(0,1)` |
| 🟡 中 | 10 帧 / 833ms | 啄-啄-停-啄 | `(0,0)(0,4)(0,1)(0,5)(0,2)(0,2)(0,2)(1,6)(0,4)(0,5)` |
| 🔴 猛 | 6 帧 / 500ms | 黑漂横拉 | `(0,0)(-3,5)(-5,10)(4,12)(-2,12)(0,12)` |

锚点为浮漂底端中心，整数像素，不做亚像素插值。
持续态：轻/中循环回帧 0；猛档保持帧 5 + 1Hz 横向 ±2px 微抖直到提竿。

### §1.4 运行时参数传递规范

1. 仅 `_drawPixelBob` 调用前一行加偏移量，函数本体禁止修改
2. clipBottom 不受影响
3. 三轴差异由帧表数值天然承载
4. 降级：模块未加载 fallback `(0, 0)`

### §1.5 信号差异度的"四重保险"

| 保险层 | 轻档 | 中档 | 猛档 |
|---|---|---|---|
| ① 浮漂抖动 | 小幅点头 | 啄停啄节奏 | 黑漂横拉 |
| ② 放大镜边框 | 无变化 | 无变化 | 边框泛红 |
| ③ 镜头微震 | 无 | 无 | 60ms 微震 |
| ④ 外周水纹 | 小波 | 中波 | 大水花溅起 |

## 2. 放大镜染色规范（v1.2 程序化重写）

### §2.1 猛档边框泛红（onBiteStart 一次性）

**绘制契约**：

```
模块：fishing/fx/MagnifierTintRenderer.js
入口：drawMagnifierTint(ctx, rect, opts)
触发：onBiteStart(强度=猛)
```

| 参数 | 值 |
|---|---|
| 蒙版色 | `#C8412B`（v3.0 通用警示红） |
| 形状 | 放大镜环形边框 4px 宽 |
| 合成模式 | `globalCompositeOperation = 'multiply'` |
| 绘制方式 | `fillRect` 拆 4 段矩形（**禁用 arc**——破坏像素硬边） |
| α 关键帧 | t=0 → 0%、t=80ms → 70%（急亮）、t=580ms → 0%（500ms 急退） |
| 缓动 | 80ms 段 linear；500ms 段 easeOutQuad |
| 整数对齐 | 所有坐标 `Math.floor()` 后绘制 |
| 复位 | 动画结束 `globalCompositeOperation = 'source-over'` |

**伪代码**：

```js
const k = [{t:0,a:0},{t:80,a:0.70},{t:580,a:0}];
ctx.save();
ctx.globalCompositeOperation = 'multiply';
ctx.globalAlpha = lerpKeyframes(elapsed, k);
ctx.fillStyle = '#C8412B';
drawRingRects(ctx, rect, 4);
ctx.restore();
```

### §2.2 LATE 警示泛红（持续呼吸至提竿）

| 参数 | 值 |
|---|---|
| 蒙版色 | `#E8553D` |
| 形状 | 同 §2.1（4px 环形 4 段 fillRect） |
| 合成模式 | `multiply` |
| α 曲线 | 2Hz 呼吸：`α = 0.30 + 0.40 * (sin(2π·t/500) * 0.5 + 0.5)`，区间 [30%, 70%] |
| 起止 | LATE 进入即开始，提竿/脱钩立即停止 |
| 配套 | 浮漂下方 8×8 ⚠ 标（程序化，由 `_drawPixelWarn(cx,cy)` 拼出） |

**8×8 ⚠ 标 ASCII 源数据**（色 `#E8553D`，描边 `#1A1A2E`）：
```
. . . X X . . .
. . X Y Y X . .
. . X Y Y X . .
. X Y Y Y Y X .
. X Y . Y Y X .   （中部空心 = 警示符竖线）
X Y Y Y Y Y Y X
X Y Y . Y Y Y X   （底点空心 = 警示符圆点）
X X X X X X X X
```
X=描边、Y=填充、.=透明

**与猛档区分**：

| 维度 | 猛档边框泛红 | LATE 警示泛红 |
|---|---|---|
| 色相 | 纯警示红 `#C8412B` | 偏橙红 `#E8553D` |
| 持续 | 一次性 500ms | 持续呼吸到提竿 |
| 节奏 | 急亮急退 | 2Hz 平稳呼吸 |
| 配套 | 镜头微震+水花 | 仅边框 + ⚠ 标 |

## 3. 镜头微震参数（仅猛档）

- **挂载点**：`cameraMicroShake`
- **触发**：`onBiteStart`（不挂 `onBiteCharge`）
- **波形**：每 10ms 在 [-2, +2] 像素范围内随机 (Δx, Δy)，60ms 共 6 次，最后归零
- **像素对齐**：随机值取整
- **影响范围**：整个游戏画布（含 UI 层）
- **频率限制**：同一咬钩窗口内 1 次

## 4. 漏提演出 800ms 分镜

### §4.1 时间轴分镜

| 时间段 | 画面元素 |
|---|---|
| 0~200ms | 浮漂快速回弹（4 帧 @ 12fps） |
| 100~600ms | 涟漪扩散（运行时参数化复用） |
| 200~700ms | 鱼影远离（远景层裁切） |
| 400~800ms | "可惜……"文案（fillText 程序化） |

### §4.2 涟漪扩散：运行时参数化复用

- **复用源**：现有 1.4s 周期 3 层错峰扩散椭圆
- **改造**：触发位置=浮漂当前位置；扩散速度×1.5；α 60%→0%；仅触发 1 个椭圆

### §4.3 鱼影远离

- **复用** `fish_shadow_3layers.png` 远景层（24×12，`#1A3A4A`，25%α）
- **运动**：浮漂下方 16px 沿 30° 斜向下，500ms 内位移 64px
- **α 衰减**：25% → 0%

## 5. PERFECT 三档（v1.2 程序化重写）

### §5.0 三档共用绘制契约

```
模块：fishing/fx/PerfectTextRenderer.js
入口：drawPerfectText(ctx, x, y, tier, t)
tier ∈ {'light','medium','heavy'}
触发：成功提竿且时机命中 PERFECT 窗口
```

**字体 fallback 链（三档统一）**：

```js
const FONT_STACK = `'Fusion Pixel 12px','Zpix','Cubic 11',
                    'Press Start 2P','Microsoft YaHei',
                    sans-serif`;
ctx.font = `bold ${fontSize}px ${FONT_STACK}`;
```

**整数像素对齐**：`x = Math.floor(rect.cx)`、`y = Math.floor(rect.cy + yOffset)`；shadowOffset/放射光线坐标 `Math.round`；`ctx.translate(0.5, 0.5)` **禁用**。

**出现/消失曲线**：

| 阶段 | t 区间 | scale | yOffset | α |
|---|---|---|---|---|
| 弹性进入 | 0 → 180ms | 0.6 → 1.15 → 1.0 | 0 | 0 → 100% |
| 停留+上飘 | 180ms → tHold | 1.0 | 0 → -8px | 100% |
| 淡出 | tHold → tEnd | 1.0 | -8 → -14px | 100% → 0 |

弹性 `easeOutBack(1.7)`；上飘 linear；淡出 easeInQuad。

### §5.1 「稳！」（轻档，单层柔光）

| 参数 | 值 |
|---|---|
| 文案 | `稳！` |

| 金光层 | **单层**：`shadowColor='#FFEEB0'`、`shadowBlur=4` |
| 时长 | 600ms（180+270+150） |
| 上飘 | -6px |

### §5.2 「妙！」（中档，双层金光）

| 参数 | 值 |
|---|---|
| 文案 | `妙！` |
| 字号 | `16px bold` |

| 时长 | 700ms（180+370+150） |
| 上飘 | -8px |

### §5.3 「完美！」（猛档，三层金光 + 4 道放射光）

| 参数 | 值 |
|---|---|
| 文案 | `完美！` |
| 字号 | `20px bold` |
| 主色 | `#FFF4D6` |
| 描边 | `lineWidth=2`、`#1A1A2E` |
| 金光层 | **三层**：① `#FFEEB0` blur 2 ② `#FFD43B` blur 6 ③ `#D4A41A` blur 12（三次 fillText 叠加） |
| 放射光 | 4 道（见下） |
| 时长 | 800ms（180+470+150） |
| 上飘 | -10px |

**4 道放射光线参数**：

| 参数 | 值 |
|---|---|
| 数量 | 4 道 |
| 角度 | 0° / 90° / 180° / 270°（+/X 形态对齐字心） |
| 长度 | 12px（固定不缩放） |
| 宽度 | 2px |
| 起点 | 距字心 8px |
| 色 | `#FFEEB0` 100%α → 外端 0%α 渐变 |
| 实现 | `linearGradient` + `fillRect` + `ctx.rotate` |
| 时序 | 与字同步 |
| 整数对齐 | rotate 后 `Math.round` 落点 |

**伪代码**：

```js
const angles = [0, 90, 180, 270];
ctx.save();
ctx.translate(cx, cy);
for (const deg of angles) {
  ctx.save();
  ctx.rotate(deg * Math.PI / 180);
  const grad = ctx.createLinearGradient(8, 0, 20, 0);
  grad.addColorStop(0, 'rgba(255,238,176,1)');
  grad.addColorStop(1, 'rgba(255,238,176,0)');
  ctx.fillStyle = grad;
  ctx.fillRect(8, -1, 12, 2);
  ctx.restore();
}
ctx.restore();
```

## 6. 四叶草微光 🍀（v1.2 程序化重写）

### §6.1 7×7 像素图（保留 v1.1，作为程序化数据源）

```
. . G G G . .
. G G H G G .
G G H H H G G
G H H H H H G
G G H H H G G
. G G H G G .
. . G G G . .
```

- `G = #7AC862` 主色绿
- `H = #F4E4BC` **四叶草专用高光**（用途收窄标注，v1.2 修订）
- `. = 透明`
- 外发光 `#FFEEB0` 4px 30%α

### §6.2 程序化绘制契约

```
模块：fishing/fx/CloverGlowRenderer.js
入口：drawCloverGlow(ctx, x, y, t)
触发：保底咬钩前 0.5s（仅放大镜内可见）
```

| 参数 | 值 |
|---|---|
| 数据源 | §6.1 7×7 ASCII（硬编码常量数组） |
| 绘制方式 | 逐像素 `fillRect(x+i, y+j, 1, 1)` |
| 锚点 | 中心 (3,3)，落点 `Math.floor(x-3)`、`Math.floor(y-3)` |
| 总帧数 | 6 帧（500ms，requestAnimationFrame 时间驱动） |

**6 帧关键帧**：

| 帧 | t (ms) | scale | α | shadowBlur | 说明 |
|---|---|---|---|---|---|
| 1 | 0 | 0.4 | 0 | 0 | 起始 |
| 2 | 80 | 1.20 | 100% | 4 | 弹性超调（最显眼） |
| 3 | 160 | 1.0 | 100% | 4 | 回稳 |
| 4 | 280 | 1.0 | 100% | 3 | 停留 |
| 5 | 400 | 1.0 | 60% | 2 | 上飘 -2px、淡出 |
| 6 | 500 | 0.9 | 0 | 0 | 上飘 -4px、消失 |

**外发光实现**：
- `ctx.shadowColor = '#FFEEB0'`、`shadowBlur = 关键帧值`、`shadowOffsetX/Y = 0`
- 仅作用于绿色 G 像素（绘制 H 时临时 `shadowBlur = 0`）

**缩放实现**：
- `ctx.translate(x, y); ctx.scale(s, s); ctx.translate(-3, -3)` 包裹
- 弹性帧子像素糊化是「弹性手感」来源，**不消除**；scale=1 帧严格整数对齐

**红线**：无文字、不上 HUD、位置 ±2px 浮动避免被识破

## 10. 关键设计决策溯源

| 决策 | 主美理由 |
|---|---|
| 三档差异化「三轴+四重保险」 | 单维差异化易被色弱/眼花玩家误读 |
| 浮漂改程序化（v1.1） | v3.0 已是 Canvas 程序化，加 Δy/Δx 偏移最稳；AI sprite 帧间精度无法保证 |
| 色板对齐 v3.0 代码（v1.1） | v1.0 误标 `#1A1A1A`、`#C8412B`、缺黄环 |
| 米灰孤立色合并 `WHT_DK`（v1.1） | grep 全工程无 `#A89888` 共鸣 |
| **D5 全程序化路线（v1.2）** | **见下：四支柱论证** |

### §10.1 v1.2 全程序化路线四支柱论证

1. **数学函数轨迹**：所有 9 项资产视觉本质 = 几何形状 + 时间函数 + 色板查表
   - 浮漂抖动 = `_drawPixelBob(cx+Δx(t), cy+Δy(t))`
   - 放大镜染色 = 4 段 fillRect + multiply + α 关键帧
   - 水花粒子 = 抛物线 `y=v₀t-0.5gt²` + N=12 粒子
   - 四叶草 = 7×7 ASCII + scale/α 关键帧
   - PERFECT 字 = fillText + shadowBlur 多层叠加
   - 放射光 = 4 道 linearGradient × fillRect × rotate
   - 「可惜」= fillText + α 双段曲线
   均可 ≤80 行代码精确表达。**数学函数 100% 复现，AI 生图无法复现**。

2. **像素硬边底线**：D5 与 v3.0 浮漂同 Canvas 渲染，AI 生图反锯齿采样产生「软-硬-软」边缘断裂；程序化 fillRect 天然 1px 硬边、零锯齿。

3. **v3.0 工程统一基准**：v1.1 已确立 12 色调色板 + `_drawPixelXxx` 系列函数管线；新资产沿用同管线 = 零学习成本、零集成风险、零额外构建。

4. **0 token 消耗**：v1.1 「8 张实际生图」预估 ~16k token + 后续每次微调重新生图 + 索引化重映射；v1.2 程序化 = 主程一次性写完模块 + 后续参数热更新 = **生图 token 0、迭代成本趋零**。

### §10.2 反向论证（不程序化的成本）

- AI 生图后人工像素清边 ≥3 轮 / 张 × 8 张 = 24 轮
- 索引化色板重映射 ≥2 轮 / 张 × 8 张 = 16 轮
- 任意参数微调需重新生图 + 重清边
- 总工时估算 ≥3 人日，**远高于程序化 ≤1 人日**

| 决策 | 主美理由 |
|---|---|
| 中档 PERFECT 「妙！」 | 语义梯度 + 单字易读 + 文化语境契合 |
| 四叶草绿+金双色 | 绿=Lucky 文化通感，金=惊喜光泽 |

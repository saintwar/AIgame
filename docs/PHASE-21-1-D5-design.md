## 1. D5 核心玩法定义

**鱼咬钩反馈系统（Bite Feedback System）** 是钓鱼场景从 `Waiting`（等鱼）切入 `BiteWindow`（咬钩窗口）瞬间起、到玩家做出"提竿/不提竿"决策为止的一段**操作博弈片段**。

**做什么**：当鱼真正咬钩时，浮漂以"轻 / 中 / 猛"三档力度抖动，玩家通过**听觉、视觉、节奏感**综合判断"现在是不是该提竿、值不值得提竿"，并在一段≥0.8s 的反应窗口内完成提竿动作。

**为什么这样设计**：
- v2.0 浮漂博弈已确立"读浮漂→判断时机→果断提竿"的玩法灵魂，D5 是该灵魂在 PHASE 21-1 v3.0 6 阶段链路中的**正式落地节点**
- 老板拍板"三档力度"而非"按鱼种"，目的是让信号**易读、易感知、不剧透**，把决策门槛降到移动端单手可操作
- 三档分级 + 暗示但不剧透，制造"这次到底是什么鱼？"的微悬念，让每次咬钩都是一次小小的赌局

**玩家体验目标（一句话）**：
> "我看着浮漂、听着水声，心里有谱但没把握——就这一下，提！"

**情绪定位**：从阶段 3 的"禅意低位"猛地拉高一格到"专注+期待"，但**不要拉满**——拉满留给 D6 提竿瞬间和 D7 搏斗。D5 是**勾引玩家入局**的钩子，不是高潮本身。

## 2. 三档力度玩法映射表（核心交付物）

| 维度 | 🟢 轻档 Light | 🟡 中档 Medium | 🔴 猛档 Heavy |
|---|---|---|---|
| **触发占比** | ~55% | ~30% | ~15% |
| **触发鱼群（主）** | 小杂鱼群 | 中型鱼群 | 罕见大鱼群 |
| **浮漂行为** | 短促小幅点头，下沉 1~2 次后稳定下沉 | 节奏感明显的连续下沉，伴有 1 次小停顿 | 一次性深沉黑漂，带横向拉拽 |
| **抖动幅度（主美参考）** | 浮漂高度变化 ~3px / 震幅小 | ~6px / 震幅中等 | ~12px+ 横向拉 4~6px |
| **音效语境（不出资源）** | "笃、笃" 清脆水声，轻巧 | "咚—咚—咚" 中频闷响，节奏感 | "嘭！哗——" 重低音 + 水花炸开 |
| **触感 Hook（移动端预留）** | 轻微短震 ×1 | 中等震 ×2（间隔 80ms） | 强长震 ×1（150ms+） |
| **画面氛围** | 浮漂周围水纹小波 | 水纹扩大、放大镜内浮漂明显抖 | 镜头 0.05s 微震 + 水花溅起 + 放大镜边框泛红 |
| **玩家应感受到** | "嗯？有动静，可能是小鱼，提一下试试？" | "来了来了，节奏稳，得抓准！" | "卧槽！大家伙！别慌别慌……" |
| **窗口长度** | 1200ms | 1000ms | 800ms（红线下限）|
| **奖励钩子（见 §2.1）** | 连击 / 手感 / 升级素材 | 标准金币 + 经验 | 稀有度奖励 + 图鉴进度 |

### 2.1【必答】轻档如何让玩家"想提"而不是"懒得提"？（Nina 张力点解法）

**机制 A：连击节奏奖励 — Combo Streak**
- 连续在轻档窗口内提竿成功 ≥3 次，触发 **"手感渐入"** 状态：下一次咬钩窗口时长 +200ms，且放大镜边框泛微金 0.5s
- 连击 5 次：触发**"渔人之眼"** 临时 buff（持续 60s），所有档位窗口 +100ms
- 连击在中/猛档完美提竿不重置，但在空竿/漏提时清零

**机制 B：副产物经济 — 边角料价值**
- 小鱼掉落"鱼鳞 / 鱼骨 / 内脏"等**升级素材**（与 D6+ 装备线挂钩）
- 例：鱼鳞 ×30 → 升级竹制竿耐久；鱼骨 ×10 → 解锁"碎冰篓"图鉴

**机制 C：低门槛 PERFECT 反馈**
- 轻档窗口 1200ms，但**完美区间放宽到 0~500ms**（最大档），完美率显著高于中/猛档
- PERFECT 触发**金光小字 + "稳！"**（区别于猛档的 "完美！"）

**总结轻档的吸引力公式**：**手感（连击）+ 资源（素材）+ 高频爽点（宽 PERFECT 区）**

## 3. 三档与已有鱼群（D1）的对应关系

### 3.1 主映射（70% 时候是这样）

| 鱼群（D1） | 主映射档位 | 占比 |
|---|---|---|
| 小杂鱼群（密集） | 轻档 | 70% |
| 中型鱼群（中密） | 中档 | 70% |
| 罕见大鱼群（稀疏） | 猛档 | 70% |

### 3.2 暗示打乱层（30% 反向洗牌，决策 2 落地）

| 鱼群 | 30% 越档分布 |
|---|---|
| 小杂鱼群 | 25% 中档 + 5% 猛档（彩蛋）|
| 中型鱼群 | 20% 轻档 + 10% 猛档（鱼群惊扰大物路过）|
| 罕见大鱼群 | 25% 中档 + 5% 轻档（潭神使者假动作）|

**玩家可读心智模型**：看到猛档 → "**大概率**大鱼，但也可能是中型鱼群里有意外"

**红线对齐**：三档力度只暗示**鱼群体型大致档位**，不绑定鱼种。具体鱼种揭晓在 D6 提竿成功后的 `FISH_REVEAL` 阶段。

### 3.3 越档收益设计

- 轻档命中却出中档/猛档鱼 → 触发 **"小鱼大物"** 横幅，金币 ×1.5
- 猛档命中却是小鱼（5%） → 不惩罚，但鱼名前缀加"机灵的"，作为图鉴趣味条目

## 4. BiteWindow 时序设计（决策 3 落地）

### 4.1 各档窗口三段切分

| 档位 | 总长 | PERFECT 完美区 | GOOD 良好区 | LATE 普通区 |
|---|---|---|---|---|
| 🟢 轻档 | **1200ms** | 0~500ms（宽，故意） | 500~900ms | 900~1200ms |
| 🟡 中档 | **1000ms** | 0~350ms | 350~750ms | 750~1000ms |
| 🔴 猛档 | **800ms**（红线下限）| 0~250ms | 250~600ms | 600~800ms |

### 4.2 与 v2.0/v3.0 既有判定的衔接

| 判定结果 | 输出给 D6/搏斗的张力起始 | 鱼 HP 修正 | fishPull 系数 |
|---|---|---|---|
| PERFECT | 50（绿区中央） | -10% | ×0.85 |
| GOOD | 60 | 标准 | ×1.0 |
| LATE | 78（贴红区） | 标准 | ×1.15 |

→ **完全沿用 v2.0 表，无需改动 BattleStateSystem**

### 4.3 连续轻档奖励的窗口加成

- "手感渐入"状态（连击 ≥3）：当前档位窗口 +200ms（加在 LATE 段尾部）
- "渔人之眼" buff（连击 5）：所有档位窗口 +100ms（加在 LATE 段尾部）
- 加成不影响 PERFECT/GOOD 区段时间，只放宽容错

## 5. 早晚提的软惩罚逻辑（决策 4 落地）

### 5.1 早提（BiteWindow 进入前的 Waiting 阶段提竿）

| 子状态 | 早提行为 | 后果 |
|---|---|---|
| IDLE（鱼未接近）| 提竿 | **空竿无惩罚**，浮漂 0.3s 拉起后回 Waiting |
| APPROACH（鱼接近中）| 提竿 | **空竿 + 鱼受惊**，本次 Waiting 重置 +2~4s |
| NIBBLE 试探阶段 | 提竿 | **空竿 + 鱼吓跑**，下一次咬钩**力度档位降 1 档** |

### 5.2 晚提（LATE 区之后到窗口关闭瞬间）

- **概率失败**：在 LATE 区段 100% 进入"风险提竿"——
  - 50% 提竿成功（进 D6 搏斗，张力 78 贴红区，首段挣扎波 +1）
  - 30% 鱼挣脱（演出"鱼吐钩"0.5s）
  - 20% 钓线打结（特殊空竿，下一次 Waiting +1s）

### 5.3 漏提（窗口完全错过）

- 窗口结束 100ms 后进入**"鱼跑掉"演出 0.8s**（涟漪 + 鱼影远离 + "可惜……"）
- 演出结束后回 Waiting，**等待时长 +3~6s 惩罚**
- 同鱼群再次咬钩概率 60%，且**下一次档位 +1 提示"补偿"**

### 5.4 累计难度叠加表

| 累计行为 | 叠加效果 |
|---|---|
| 连续 2 次早提（NIBBLE 阶段）| 鱼群整体警觉 +1，下次咬钩档位降 1，BiteWindow 总长 -100ms |
| 连续 3 次漏提 | 触发 PRD 保底进度 |
| 早提 + 漏提混合连续 5 次 | 进入"运气太背"补偿——下一次强制 GOOD 区段宽度 +30% |

**所有惩罚显式可视化**：屏幕底部进度条小图标变红/变灰。

## 6. PRD 保底机制接入（决策 5 落地）

### 6.1 触发条件（5+1）

- 维护 `consecutiveFailCount` 计数器（D5 持有），失败定义包括：
  - 早提空竿（NIBBLE/APPROACH 阶段提竿）
  - 漏提（窗口完全错过）
  - 晚提失败（30% 鱼挣脱、20% 钓线打结）
  - **不计入失败**：IDLE 早提、晚提成功、PERFECT/GOOD 命中

- 当 `consecutiveFailCount == 5`，下一次咬钩**强制触发保底**

### 6.2 保底咬钩规格

- **档位**：**强制中档**（避免猛档加倍受挫，避免轻档信号弱被忽略）
- **窗口**：1500ms（中档基准 1000ms + 保底加成 500ms），PERFECT 区放宽到 0~600ms
- **鱼种**：从中型鱼群剔除最难搏斗的 erratic 行为，保留温和的吴郭鱼，fishPull ×0.85

### 6.3 是否给玩家暗示"这次是保底"

**给暗示，但用"幸运感"语言而非"系统补偿"语言**：

- 浮漂咬钩前 0.5s 出现 **🍀 四叶草微光**（放大镜内一闪而过，0.3s）
- 抖动音效叠加一层"叮"的清脆铃声（与普通中档区分）
- **不写"保底""补偿""新手"任何字眼**

### 6.4 保底成功后

- 计数器清零
- 不连发保底（保底命中后再失败 5 次才会再次触发）
- 保底命中**不计入连击**

## 7. 音效/触感预留接口清单

### 7.1 浮漂事件 Hook

| Hook 名 | 触发时机 | 主要消费方 |
|---|---|---|
| `onBiteApproach` | Waiting → BiteWindow 切换前 200ms | 音效 + 浮漂微动画 |
| `onBiteStart(level)` | BiteWindow 进入瞬间 | 主美抖动动画 + 主程窗口计时器启动 |
| `onBiteCharge(level, t)` | BiteWindow 内每帧 | 音效节奏 + 触感渐进 |
| `onBitePerfectZone` | 进入 PERFECT 区段瞬间 | 极轻提示音"叮" |
| `onBiteLateZone` | 进入 LATE 区段瞬间 | 浮漂边框微泛红 |
| `onBiteRelease(result)` | 玩家提竿 | 决定后续演出分支 |
| `onBiteWindowClose` | 窗口超时关闭 | 触发漏提演出 |
| `onBitePenaltyApply(type, level)` | 软惩罚生效瞬间 | UI 红字提示 + 计数器 |
| `onBiteLuckyHint` | 保底咬钩前 0.5s | 四叶草特效 + 铃声 |

### 7.2 触感（Haptic）预留

- 仅移动端实装，PC/Web 端忽略
- 三档对应：`HAPTIC_LIGHT_TAP` / `HAPTIC_MEDIUM_DOUBLE` / `HAPTIC_HEAVY_LONG`

### 7.3 视觉效果挂载点

- `floatShakeSlot`：浮漂抖动动画播放槽位（主美 3 套动画）
- `magnifierFrameTint`：放大镜边框染色槽位（轻=无、中=无、猛=泛红）
- `cameraMicroShake`：仅猛档触发，振幅 2px / 60ms

## 8. D5 与 D6 的接口契约

### 8.1 D5 → D6 输出事件

| 事件 | 时机 | Payload |
|---|---|---|
| `BITE_DETECTED` | BiteWindow 进入瞬间 | `{ timestamp, level, fishGroupRef, isLuckyBail, windowDuration, perfectZone, goodZone, lateZone }` |
| `BITE_PERFECT_HIT` | PERFECT 区提竿 | `{ timestamp, level, reactionTime }` |
| `BITE_GOOD_HIT` | GOOD 区提竿 | `{ timestamp, level, reactionTime }` |
| `BITE_LATE_HIT` | LATE 区提竿（成功）| `{ timestamp, level, reactionTime, riskFlag: true }` |
| `BITE_LATE_FAIL` | LATE 区提竿（失败 3 类）| `{ timestamp, level, failType: 'throw'|'knot' }` |
| `BITE_MISSED` | 漏提 | `{ timestamp, level, fishGroupRef }` |
| `BITE_EARLY_FIRED` | 早提 | `{ timestamp, subState }` |
| `HOOK_JUDGED` | **D5→D6 核心交接事件** | `{ judgement: 'PERFECT'|'GOOD'|'LATE', initialTension: 50|60|78, hpModifier, fishPullCoef, fishGroupRef, biteLevel, isLuckyBail }` |

### 8.2 状态归属

| 状态 | 归属 |
|---|---|
| BiteWindow 计时器 | **D5 持有** |
| 三档力度生成与抽样 | **D5 持有**（BiteLevelDispatcher）|
| 连击计数 / 保底计数 / 软惩罚累计 | **D5 持有**（BiteFeedbackContext）|
| 提竿判定 PERFECT/GOOD/LATE | **D5 持有** |
| 搏斗起始张力/HP/系数 | D5 计算后通过 `HOOK_JUDGED` 传递给 D6 消费 |
| 鱼具体鱼种抽样 | **D6 接管**（避免 D5 提前剧透）|
| 搏斗状态机 | **D6/BattleStateSystem 接管** |

## 9. 5 人小测验收 DoD

### 9.1 玩家**应该**感受到的 3 个核心体验点 ✅

1. **"我能看出大致是大鱼还是小鱼，但每次都不太确定到底是哪一条"** — 决策 1+2
2. **"提小鱼也挺爽的，节奏抓对了会有奖励感，根本停不下来"** — Nina 张力点 §2.1
3. **"提竿时机我能掌握，不会觉得'反应不过来'，但太慢也会出问题"** — 决策 3+4

### 9.2 玩家**不应该**感受到的 2 个反例 ❌

1. **"全是小鱼，太无聊了，想跳过"**
2. **"力度看不清楚，到底轻档和中档有啥区别？"**

### 9.3 量化指标

- 5 人小测中**每人在 30 分钟内至少触发 1 次保底**
- 平均 PERFECT 命中率：轻档 ≥40% / 中档 ≥25% / 猛档 ≥15%
- 主动提竿率（玩家见到轻档抖动 → 提竿）≥80%
- 5 人中至少 4 人能口头描述出"轻/中/猛"三档差异

## 11. 设计决策溯源

| 决策 | 理由 |
|---|---|
| 猛档窗口取下限 800ms | 紧扣红线、配合"强反馈+短窗口"心理学 |
| 轻档窗口 1200ms 而非更长 | 再长会让"轻档=送分题"，损害紧张感 |
| 三档主映射 70%+越档 30% | 70% 保证可学习性，30% 保留惊喜 |
| 漏提加 +3~6s 等待 | 让"错过"有重量感 |
| 保底用中档而非猛档 | 大鱼初体验留给玩家"主动钓到"的成就感 |
| 连击不让保底参与 | 保底是体验保险，连击是技术奖励，两者不能混 |

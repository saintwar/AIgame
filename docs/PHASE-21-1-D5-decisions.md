## B 决策（玩法）：BiteLevelDispatcher 映射 key — 选 **B3 + 70/30 越档扰动**（主策划拍板）

### 核心规则（必须严格按此实施）

1. **主映射**：基于"将给玩家的这条鱼"的 `rarity`（1-5★）
   - `1-2★ → light`（轻档主档）
   - `3★ → medium`（中档主档）
   - `4-5★ → heavy`（重档主档）

2. **越档扰动层**（防破解，必须保留）：每次主档出来后按以下表扰动出**最终档**
   - 主档 light：70% light / 25% medium / 5% heavy
   - 主档 medium：70% medium / 15% light / 15% heavy
   - 主档 heavy：70% heavy / 25% medium / 5% light
   - （越档比例与 D5 PRD §3 既定的 70%+30% 越档机制一致，不要自创比例）

3. **实施落点**：`BiteLevelDispatcher.resolve(fishRarity)` 内完成
   - 输入：`fishRarity`（1-5★，由上游 FishGroupSystem 抽鱼时传入）
   - 输出：`biteLevel`（light/medium/heavy 三档之一，已含越档扰动）
   - **不侵入 FishGroupSystem**（不加 `densityType` 字段）

4. **HOOK_JUDGED 事件 payload**：`biteLevel` 字段为最终扰动后的档位
   - 鱼种 roll 时机不变（D6 接管这条红线不动）

### 玩法理由（主策划原话精简）

> D5 老板 5 决策已定调"暗示但不精确剧透"。"信号档位与鱼种解耦"指的是**分布概率不能由鱼种倒推**，但浮漂抖动幅度的体感**必须**绑定"这一杆钓到的具体这条鱼的体量"——否则在稀有鱼群里钓到 1★ 杂鱼浮漂猛抖，就违反 PRD 的"老虎机微悬念"原则。**惊喜来自每一杆，不是来自钓点。**

## 1. hotfix-z3 钓鱼场景任务 HUD 误显示

### 现象
进钓鱼场景后，画面顶部出现村庄主线任务的"q003 钓上 3 条鱼 0/3"HUD 条，跟战斗 UI 抢视觉焦点。

### 根因
`village-scene.js` 的任务 HUD 默认 params 里硬编码了 `target: 3`，导致钓鱼场景渲染共享 HUD 时也判定有任务在跟踪。

### 修复
默认参数不再硬编码 `target`，让上层根据当前场景按需传入。钓鱼场景不传 → HUD 不渲染。

**文件**：`js/village-scene.js`

## 2. hotfix-z4 教学弹窗图示简化

### 改动
拉力条说明文字 + 配图从"3 段三色（红绿红）"改为"2 段双色"——和实际拉力条只关心"安全 / 危险"两态对齐，避免新手误以为有 3 个不同区。

加一根绿色箭头从说明文字指向画面右下角的 HUD，明确"看右下"。

**文件**：`js/fishing-scene.js`（教学层 `_renderTutorialOverlay` 系列）

## 3. hotfix-z4-2 教学红箭头闪烁穿帮修复

### 现象
红箭头是"闪烁吸引注意"的，但实现时整个箭头（含黑色描边）一起做透明度动画，闪到 alpha=0.3 时黑边变浅，露出后面木牌底色，看起来像箭头"破了一道缝"。还出现过一帧黑红尖端坐标算不一致，箭头尖出现错位。

### 修复
- **黑色描边线保持 alpha=1.0**，只让红色填充做闪烁
- 黑红箭头的尖端坐标用同一个变量计算，杜绝错位

**文件**：`js/fishing-scene.js`

## 4. hotfix-z5 / z5b 抛投落点圈颜色三连改

| 版本 | 颜色 | 反馈 |
|------|------|------|
| 原版 | `#00BFFF` 青蓝 | 在浅水滩 / 绿水里对比度差，看不清 |
| z5 | `#FF3B30` 纯红 | 太刺眼，像报错弹窗 |
| **z5b（最终）** | **`#FF6FB5` 粉红** | 醒目但柔和，水面背景下识别度高 |

填充 alpha 0.28 / 外圈 alpha 0.95 / 中心十字保留不变。

**文件**：`js/systems/cast-aim-system.js`
**常量**：`LANDING_COLOR`
